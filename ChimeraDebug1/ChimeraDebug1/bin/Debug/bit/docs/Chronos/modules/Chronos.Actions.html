Module Chronos.Actions

Actions and failures are recorded for each test case and test suite as you run checks. These functions allow you to programmatically add your own actions and failures. 

 TestCase "My Test Case"
 {
   function()
     AddAction( "Running important code" )
     if not ImportantCode() then
       AddFailure( "Important code failed" )
     end
   end
 }

Functions

AddAction (message, options) Adds an action message to the recorded actions of the current test scope.
AddFailure (message, level) Adds a failure message to the recorded failures of the current test scope.


Functions

AddAction (message, options)
Adds an action message to the recorded actions of the current test scope. This function is exported into the global table.

Parameters:

  • message: The message to record.
  • options: Additional options. (Optional) { Expected = value, Actual = value }

Return value:

    A fully-formed Action, if there was an active recorder.
AddFailure (message, level)
Adds a failure message to the recorded failures of the current test scope. This function is exported into the global table.

Parameters:

  • message: The message to record.
  • level: The error level of the failure. (Optional) See Lua error()

Return value:

    A fully-formed Failure, if there was an active recorder.
