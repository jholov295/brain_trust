Module Chronos.Auto

A typical usage of AutoChronos looks like this: 

 require "Chronos"
 require "Chronos.Auto"
 require "gScripted"
 
 local deviceComm
 
 TestSuite "Auto-Generated Test Cases"
 {
   	Auto "commandsets/Audi_KWP.xml"
   	{
   		{ Pattern = "Coding Block .*", Reqs = { "SR-123" } },
   		"ALS Return Pulse Width Values",
   		{   			Name = "Aim Calibration Values",
   			Params =
   			{
   				["First Row"] = { Valid = { 6, 10, 55 }, Invalid = { 1, 99 } },
   				["First Column"] = { Valid = { 50 }, Invalid = { 107 } },
   				["Retrieve_Stored_Error"] = { Valid = { 1 } }
   			},
   			Verify = true
   		}
 
   		Setup = function()
   			deviceComm = DeviceComm.new( "commandsets/Audi_KWP.xml" )
   		end,
 
   		Test = function( command )
   			local results = AutoRun( deviceComm, command )
   			AutoCheck( deviceComm, results )
   		end
   	}
 }
Important Notes
  • You can pass in the regular TestCase functions, Setup, Teardown, and Test.
  • The Test function here is different in that it takes an argument. AutoChronos will do some preprocessing and call this function with a Command object, set up according to what configuration options were passed in.

Command Descriptions
In a call to Auto, the user is expected to provide a path to a command set and any number of Command Descriptions. Command Descriptions can be a simple string, which will be interpreted as the name of the Command to run, or it can be a table of explicit configuration options.

Usage:

 Auto "commandsets/mycommands.cmdset"
 {
  	"Command 1",
  	{ Name = "Command 2" },
  	{
  		Name = "Command 3",
  		Reqs = { "SR-123", "SR-456" },
  		Mode = "Write",
  		Verify = true,
  		Params =
  		{
  			Param1 = { Valid = { 1, 2, 3 }, Invalid = { 4, 5, 6 } },
  			ArrayParam = { Valid = { { 1, 2 }, { 3, 4 } }, Invalid = { { 4, 5 } } },
  		}
  	},
  	{ Pattern = "Command %d", Mode = "Do" },
 
  	Test = ...
 }


Fields:

  • Name: {string} The name of the command.
        Either Name or Pattern must be specified.
        Specifying a nonexistent command is an error.
  • Pattern: {string} A pattern used to pull multiple commands from the command set and apply these options to all of them.
        Either Name or Pattern must be specified.
        This option overrides Name.
  • Reqs: {table} [OPT] A table of strings used to link requirements to generated test cases.
  • Mode: {string} [OPT] The mode to use for this command. ("Do", "Write", or "Read".)
        Only needed if more than one mode is available.
        If omitted, all available modes will be used.
        Specifying an unavailable mode is an error.
  • Verify: {boolean} If true, AutoCheck will run the command in Read mode and check that the returned values match those written in AutoRun.
        Only valid on commands that are both writable and readable.
        Requires that Mode is set to Write.
  • Params: {table} A table of command parameters, each with subtables of Valid and Invalid values.
        Specifying this table, even if it is empty, prevents automatic generation of parameters.
        Therefore, if you use Params to specify values for a single parameter, the other parameters will not be tested for anything other than their default values.
        Specifying nonexistent parameters is an error.

  • Test: The Test function here is different in that it takes an argument. AutoChronos will do some preprocessing and call this function with a Command object, set up according to what configuration options were passed in..
  • Setup: The setup function. Called before the Test function.
  • Teardown: The teardown function. Called after the Test function. This is called even if the Test fuction fails.

Functions

Auto (commandSet)
Creates an AutoChronos test. This will create test cases automatically and try to cover most if not all Commands in a CommandSet. Use this instead of the TestCase function.

Parameters:

  • commandSet: {string} The path to a CommandSet to be used to generate the test cases.

See also:

  • Command Descriptions
AutoCheck (deviceComm, results)
This is the "Read" step of the test. If the command under test has Verify = true (defaults to true for commands that are both writable and readable), then this step will do a Read and then a CheckEqual to make sure that the returned values match the written values.

Parameters:

  • deviceComm: {table} The DeviceComm object to use to run the tests.
  • results: {table} The results returned from AutoRun. This contains all the results from a previous run.

See also:

  • Command Descriptions
AutoRun (deviceComm, command)
The "Write" step of the test. The actual run mode may be "Do", "Write", or "Read". This is typically used in the Test() function, in the definition of an AutoChronos test case.

Parameters:

  • deviceComm: {table} The DeviceComm object to use to run the tests.
  • command: {table} The Command to run.
